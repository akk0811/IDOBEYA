# IDOBEYA ユーザー側デザイン案

## コンセプト

### Warm Room UI

IDOBEYA は、投稿が流れていくSNSではなく、目的ごとの「部屋」に人が集まるSNSとして設計する。

画面を開いた瞬間に見せたいのは、刺激的なトレンドや強い通知ではなく、「今日はどの部屋なら話せそうか」を自然に選べる空気。

デザインの中心は、次の3つ。

- 部屋を探しやすい
- 入室の心理的ハードルが低い
- 長く読んでも疲れない

## 既存SNSからの学び

### 採用する要素

Discord からは、目的別の空間に人が集まる構造を採用する。  
ただし、黒背景・高密度・ゲーマー感は避ける。

LINEオープンチャットからは、匿名性、部屋ごとのプロフィール、参加前の安心材料を採用する。  
部屋に入る前に「ここは自分がいてもよさそう」と感じられる設計にする。

Reddit からは、部屋ごとに文脈があり、投稿が会話の起点になる構造を採用する。  
ただし、投票やランキングによる競争感は弱める。

Instagram Stories からは、完璧な投稿でなくても気軽に共有できる感覚を採用する。  
ただし、映えやフォロワー競争には寄せない。

### 避ける要素

TikTok のような無限スクロール、強い没入、アルゴリズム主導の消費体験は避ける。

X のようなリアルタイム拡散、強いトレンド、議論の加熱を促す見せ方は避ける。

Instagram Reels のようなクリエイター競争、見た目の派手さ、動画中心の刺激設計は避ける。

## デザイン方針

### 1. フィードではなく部屋を主役にする

ホーム画面は、投稿一覧ではなく「部屋を選ぶ場所」として設計する。

おすすめの部屋、最近あたたかく動いている部屋、初心者でも入りやすい部屋を中心に見せる。

### 2. 数字は煽らず、安心材料として扱う

参加人数、投稿数、コメント数は強調しすぎない。

数字は「人気の証明」ではなく、「人がいて会話が成立している安心材料」として見せる。

### 3. 入室前に雰囲気を伝える

部屋カードには、名前・説明・参加人数だけでなく、雰囲気を表すバッジを付ける。

例:

- 初心者歓迎
- 見るだけOK
- 相談しやすい
- ゆっくり会話
- 承認制
- 匿名OK

### 4. 通知と反応は控えめにする

通知は急かさない。  
未読は薄いベージュ背景、重要通知のみアクセントカラーを使う。

いいね数やコメント数は投稿カードの下部に小さく表示し、承認欲求を刺激しすぎない。

### 5. 投稿作成は低圧にする

投稿画面は「投稿しなければならない」ではなく、「少し話してみる」感覚にする。

文言はやわらかくする。

- なにか話してみよう
- 今の気持ちを少しだけ
- 見るだけでも大丈夫
- 匿名で話す

## ビジュアル案

### カラー

基本色は既存指示を採用する。

- 背景: `#FAF8F5`
- カード: `#FFFFFF`
- メイン: `#5F8D6B`
- メイン濃色: `#4D7358`
- アクセント: `#D97706`
- 本文: `#333333`
- 補足: `#777777`
- 枠線: `#E8E1D8`
- 危険操作: `#D9534F`

グリーンは「選択中」「参加中」「投稿」など、安心して進む行動に使う。  
オレンジは「盛り上がり」「重要なお知らせ」など、ごく限定的に使う。

### 形

カードは角丸 `16px` 相当。  
ボタンは丸みを強くし、押しやすく、強すぎない印象にする。  
投稿・通知・部屋カードは、余白を広めに取り、詰め込み感を避ける。

### 影

影は軽く、浮きすぎない。

カードは「机の上に紙が置いてある」程度の柔らかさにする。

### アイコン

線画アイコンを基本にする。  
部屋アイコンだけは、部屋の温度感を出すために絵文字または丸背景のシンプルアイコンを使う。

## 5画面の具体案

### ホーム

役割: 今日行く部屋を選ぶ画面。

構成:

- 上部挨拶
- 通知ボタン
- おすすめの部屋
- 初めてでも入りやすい部屋
- 最近あたたかく動いている部屋

デザイン:

- 投稿より部屋カードを主役にする
- カードには部屋アイコン、部屋名、短い説明、参加人数、雰囲気バッジを表示
- 盛り上がりは炎で煽るより、「会話中」「今日も動いています」のように穏やかに表現する

### 探す

役割: 自分に合う部屋を探す画面。

構成:

- 検索バー
- カテゴリータブ
- 雰囲気フィルター
- 部屋一覧

デザイン:

- 検索バーは角丸で薄い背景
- カテゴリーはグリーン下線
- 部屋一覧は縦カード
- 右矢印は控えめに表示
- 「人気」だけでなく「相談」「生活」「学び」「ゆっくり」など目的軸を見せる

### 部屋

役割: 安心して会話する画面。

構成:

- 戻るボタン
- 部屋アイコン + 部屋名
- 参加人数
- 参加中バッジ
- タイムライン / メンバー
- 投稿一覧
- 下部投稿欄

デザイン:

- ヘッダーで部屋の雰囲気を明確にする
- 参加中バッジはグリーン背景
- 投稿カードは白背景、読みやすい余白
- いいね・コメント数は小さく、補足情報として表示
- 投稿欄は「話してみる」ための低圧な導線にする

### 投稿作成

役割: 気軽に投稿する画面。

構成:

- キャンセル
- 新しい投稿
- 投稿ボタン
- 投稿先の部屋
- テキスト入力
- 画像 / アンケート / 絵文字 / その他
- 匿名投稿切替

デザイン:

- 入力欄を広く取る
- 投稿ボタンはグリーン
- 補助機能はアイコン + 短いラベル
- 匿名切替は安全機能として見せる
- 「気軽に投稿してください」よりも、さらに自然な「なにか話してみよう」を使う

### 通知

役割: 反応やお知らせを落ち着いて確認する画面。

構成:

- 通知タイトル
- カテゴリータブ
- 通知一覧

デザイン:

- 通知ごとに区切り線
- 未読は薄いベージュ背景
- 重要通知のみアクセントカラー
- いいねやコメント通知は控えめにする
- 時刻は補足文字色で小さく表示

## コピー方針

全体の文言は、命令形や煽りを避ける。

避ける:

- 今すぐ参加
- 急上昇中
- 人気爆発
- 投稿して注目を集めよう

使う:

- この部屋をのぞく
- 会話を見てみる
- 今日も会話中
- ゆっくり話せる部屋
- 見るだけでも大丈夫

## 実装優先度

### Phase 1

ユーザー側5画面の静的UIを `Warm Room UI` に統一する。

- ホーム
- 探す
- 部屋
- 投稿作成
- 通知

### Phase 2

部屋カードに安心材料を追加する。

- 雰囲気バッジ
- 入室方式
- 見るだけOK
- 匿名OK
- 最終会話時刻

### Phase 3

データ連携時に、APIから受け取る情報をUIに自然に反映する。

- 部屋一覧
- 投稿一覧
- 通知一覧
- 参加状態
- 未読状態

## 最終デザインの方向性

IDOBEYA のユーザー側UIは、「SNSを開く」よりも「自分に合う部屋をのぞく」感覚に寄せる。

見た目は温かく、操作はシンプルに、反応は控えめに。  
ユーザーが開いた瞬間に、次のように感じられることをゴールにする。

「ここなら、少し話してみてもいいかもしれない」
